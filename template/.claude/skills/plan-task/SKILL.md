# 目的

将 spec 转化为可执行的工程计划，确保动手前已充分理解需求和影响范围。

# 步骤

1. 读取 `specs/active/` 下的目标 spec
2. 识别范围、约束、依赖和影响文件
3. 检查 `wiki/` 中是否有相关架构或模块文档可参考
4. 在 `plans/` 下创建计划文件，命名：`YYYY-MM-DD-任务名.md`
5. 计划必须包含：
   - 目标和非目标
   - 影响的文件/模块
   - 实现步骤
   - 测试策略
   - 风险和回退方案

# 约束

- 不在计划阶段修改 src/ 或 tests/
- 计划完成后等待用户确认再进入实现
