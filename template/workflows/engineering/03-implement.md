# Implement 工作流

按照 spec + plan 执行实现，产出代码、测试和文档三件套。

## 步骤

### 1. 确认输入

- 读取 `specs/active/` 中的 spec
- 读取 `plans/` 中的对应计划
- 确认计划已获用户确认

### 2. 实现代码

按计划中的步骤逐步实现：
- 在对应模块目录中创建或修改代码文件（如 `modules/xx/src/Matlab/`）
- 遵循 `.claude/rules/engineering.md` 中的代码规范
- 保持函数小而可测试

### 3. 编写测试

- 在模块目录中添加或更新测试文件（如 `test_xxx.m`）
- 覆盖正常路径和边界条件
- 如果不需要测试，在 plan 中说明原因

### 4. 更新模块 README

实现完成后，更新模块目录下的 `README.md`，包含：

| 章节 | 内容 | 必须 |
|------|------|:----:|
| 模块简介 | 一段话说明职责和在链路中的位置 | 是 |
| 对外接口列表 | 每个函数：参数表（方向、名称、类型、含义、默认值） | 是 |
| 内部函数接口列表 | 辅助函数，标注"不建议外部直接调用" | 否 |
| 核心算法技术描述 | 算法原理、关键公式（LaTeX）、参数选择依据、适用条件与局限性 | 是 |
| 使用示例 | 可直接运行的代码片段 | 是 |
| 依赖关系 | 上游/下游模块 | 是 |
| 测试覆盖 | 测试项表格：编号、名称、断言条件、说明 | 是 |

## 产出物

完成本步骤后应有三件套：
- [ ] 代码文件（新增或修改）
- [ ] 测试文件（新增或修改）
- [ ] 模块 README（已更新）

产出完成后进入 **04-validate** 收尾。
